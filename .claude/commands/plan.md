Plan a new feature or change before implementing:

1. Explore the codebase to understand current architecture
2. Read BACKEND_PLAN.md and CLAUDE.md for context
3. Propose 2-3 approaches with trade-offs
4. Get user approval on the approach
5. Write a detailed plan with exact file paths and code snippets
6. Each task should be small and focused

Always check existing patterns before proposing new ones.
Reference the data contracts (Company, DemoEvent) to ensure compatibility.
